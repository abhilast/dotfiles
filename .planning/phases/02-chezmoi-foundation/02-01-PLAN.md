  <action>
Create /Users/ab/dotfiles/.chezmoi.toml.tmpl with exactly this content:

```
{{- $hostname := .chezmoi.hostname -}}
{{- $arch := .chezmoi.arch -}}
{{- $name := "Abhilash" -}}
{{- $email := promptStringOnce . "email" "Git email address" -}}
{{- $isWork := promptBoolOnce . "isWork" "Is this a work machine" -}}
{{- $workspacePath := promptStringOnce . "workspacePath" "Worktree workspace path (e.g. ~/repos/skates)" -}}
{{- $signingKey := promptStringOnce . "signingKey" "Git signing key ID (GPG fingerprint or SSH key path, leave blank to skip)" -}}

[data]
  hostname = {{ $hostname | quote }}
  arch = {{ $arch | quote }}
  name = {{ $name | quote }}
  email = {{ $email | quote }}
  isWork = {{ $isWork }}
  workspacePath = {{ $workspacePath | quote }}
  signingKey = {{ $signingKey | quote }}
```

Notes on this file:
- `promptStringOnce`/`promptBoolOnce` prompts ONCE on first `chezmoi init`, stores the answer in `~/.config/chezmoi/chezmoi.toml` for subsequent runs — user is never re-prompted
- `$name` is hardcoded (not prompted) because it never varies across this user's machines — per CONTEXT.md auto-detect decision
- `$hostname` and `$arch` are auto-detected from `.chezmoi.hostname` and `.chezmoi.arch` — no prompt needed
- 1Password section is intentionally omitted from this phase — it is a v2/Phase 5+ concern
- No `[diff]` or `[git]` sections in this phase — keep minimal
- Whitespace-trimming `{{-` dashes prevent blank lines in the rendered output
  </action>
